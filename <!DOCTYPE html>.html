When HTML5 has become a standard? What were the main reasons for developing new standard?

HTML5 became a standard on October 28, 2014, by the World Wide Web Consortium (W3C). The main reasons for developing this new standard were to address the shortcomings of previous HTML versions, accommodate new technologies like multimedia elements without the need for additional plugins, improve accessibility, and facilitate the development of richer and more interactive web applications.

What is a difference between W3C HTML5 and WHATWG HTML Living Standard?

The main difference between W3C HTML5 and WHATWG HTML Living Standard lies in the development process. W3C HTML5 was developed by the W3C working group, while WHATWG HTML Living Standard was developed by the Web Hypertext Application Technology Working Group (WHATWG). W3C HTML5 is considered a snapshot of WHATWG HTML Living Standard, as it was frozen at a certain point in development.

What is the minimal HTML page code?

The minimal HTML page code is as follows:

1. Start with an opening HTML tag: "html"

2. Inside the HTML tag, include an opening head tag: "head"

3. Within the head tag, include an opening title tag: "title"

4. Inside the title tag, provide a title for the webpage (e.g., "My Webpage")

5. Close the title tag: "/title"

6. Close the head tag: "/head"

7. Open the body tag: "body"

8. Inside the body tag, add the content of your webpage (e.g., text, images, links)

9. Close the body tag: "/body"

Comment on semantic and presentational markup. Why presentational markup is considered deprecated now?

Semantic markup focuses on the meaning and structure of content, making it easier for browsers, search engines, and assistive technologies to understand and interpret the content correctly. Presentational markup, on the other hand, deals with styling and appearance, which can lead to less accessible and maintainable code. Presentational markup is considered deprecated now because it mixes content with styling, making it harder to maintain, less accessible, and less adaptable to different devices and user preferences.

How you can check browser support for particular tag or attribute?

To check browser support for a particular tag or attribute, you can refer to resources like MDN Web Docs or Can I Use website. These platforms provide detailed information about browser compatibility for HTML elements, attributes, CSS properties, and JavaScript APIs.

What are the audio and video formats, officially supported by HTML standard? What happens if you use other format?

The audio and video formats officially supported by the HTML standard are MP3 for audio and MP4 for video. If other formats are used, it might not be supported by all browsers and might require additional plugins or fallback mechanisms to play properly.
